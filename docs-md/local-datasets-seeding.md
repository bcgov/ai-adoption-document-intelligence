# Local-folder dataset seeding

Drop datasets into the repo under `data/datasets/<name>/{public,private}/` and
they are seeded into the database and synced to blob storage automatically on a
local/dev backend start. This lets you evaluate workflows against your own
sample documents without touching production data or wiring up an upload flow.

The feature has three pieces:

| Piece | File | Role |
|-------|------|------|
| Parser | `apps/backend-services/src/seed/local-datasets.ts` | Pure function that scans `data/datasets/` and returns `ParsedLocalDataset[]`. No DB or filesystem mocks required to test. |
| Seed | `seedLocalDatasets()` in `apps/shared/prisma/seed.ts` | Idempotently upserts `Dataset` + `DatasetVersion` rows from the parsed folders. Runs after the benchmark-data seed. |
| Blob sync | `apps/backend-services/src/seed/local-dataset-sync.service.ts` | Dev-only `OnApplicationBootstrap` service that copies the local files into blob storage so the benchmark / sample-preview infrastructure can find them. |

## Folder convention

```
data/datasets/
  <dataset-name>/
    public/          # visibility = public
      manifest.json
      documents/…
      ground-truth/…
    private/         # visibility = private
      …
```

`<dataset-name>` becomes the dataset's folder key. Each `public/` or `private/`
subfolder is seeded as its own dataset (visibility is taken from the subfolder
name). Either or both visibilities may be present.

## Two layouts

The parser accepts two layouts per visibility folder:

### 1. Manifest-driven (`manifest.json`)

Authoritative when you curate the file yourself. Shape:

```jsonc
{
  "datasetName": "SDPR monthly reports",   // optional; falls back to folder name
  "templateModelKey": "seed-sdpr-monthly-report-template", // optional link to a template
  "metadata": { },                          // optional free-form
  "samples": [
    {
      "id": "sample-001",
      "inputs":      [{ "path": "documents/001.jpg",        "mimeType": "image/jpeg" }],
      "groundTruth": [{ "path": "ground-truth/001.json",    "format": "json" }],
      "metadata": { }
    }
  ]
}
```

### 2. Flat-pair

Drop `<name>.<image-ext>` + `<name>.json` pairs directly into the visibility
folder. The seed calls `deriveManifestFromFlatPairs()` to auto-generate a
`manifest.json` with `metadata.autoGenerated = true`, so subsequent re-seeds
refresh it. If you remove the `autoGenerated` marker (or replace it with your
own curated manifest), the file is treated as user-managed and never
regenerated.

## What the seed creates

`seedLocalDatasets()`:

- Skips quietly with a log line when `data/datasets/` is empty — a clean
  checkout with no local datasets seeds nothing.
- Upserts one `Dataset` per folder+visibility and one `DatasetVersion` per
  dataset, using deterministic ids from `localDatasetId()` /
  `localDatasetVersionId()` so re-seeds are idempotent (no duplicates).
- Records the `storagePrefix` / `manifestPath` pointing at the *eventual* blob
  storage location; the blob-sync service (below) puts the actual files there.

## Blob sync on startup

`LocalDatasetSyncService` runs on application bootstrap and copies the local
files into blob storage so the rest of the platform (which reads documents from
blob storage) can serve them.

- **Dev-only.** Disabled automatically when `NODE_ENV === "production"`.
- **Idempotent.** Skips files already present in blob storage; a re-run after
  adding new samples uploads only the new ones. It then updates each
  `DatasetVersion`'s `storagePrefix` / `manifestPath` / `documentCount`.

### Environment flags

| Variable | Default | Effect |
|----------|---------|--------|
| `NODE_ENV` | — | `production` disables the sync entirely. |
| `SYNC_LOCAL_DATASETS_ON_START` | `true` | Set to `false` to skip the startup sync (the DB seed still runs). |
| `FORCE_RESYNC_LOCAL_DATASETS` | `false` | Set to `true` to re-upload every file even if it already exists in blob storage. |

## Typical workflow

1. Create `data/datasets/my-set/public/` and drop in either a curated
   `manifest.json` (+ `documents/`, `ground-truth/`) or flat `<name>.jpg` +
   `<name>.json` pairs.
2. Start the backend locally. `seedLocalDatasets()` creates the dataset rows and
   `LocalDatasetSyncService` uploads the files to blob storage.
3. The dataset is now selectable by the benchmark / evaluation infrastructure.
