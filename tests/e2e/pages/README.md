# Page Object Models

This directory contains Page Object Models (POMs) for Playwright tests.

## What are Page Objects?

Page Objects are TypeScript classes that encapsulate page structure and interactions. They:
- Define locators for page elements
- Provide methods for common interactions
- Make tests more readable and maintainable
- Reduce code duplication

## Structure

Each Page Object should:
1. Import `Page` and `Locator` from `@playwright/test`
2. Define readonly locators for all interactive elements
3. Accept a `Page` instance in the constructor
4. Provide methods for complex interactions

## Example

```typescript
import { Page, Locator } from '@playwright/test';

export class BenchmarkPage {
  readonly page: Page;
  readonly createButton: Locator;
  readonly nameInput: Locator;
  readonly submitButton: Locator;
  readonly successMessage: Locator;

  constructor(page: Page) {
    this.page = page;
    this.createButton = page.locator('[data-testid="create-benchmark"]');
    this.nameInput = page.locator('input[name="benchmarkName"]');
    this.submitButton = page.locator('button[type="submit"]');
    this.successMessage = page.locator('.toast-success');
  }

  async createBenchmark(name: string) {
    await this.createButton.click();
    await this.nameInput.fill(name);
    await this.submitButton.click();
    await this.successMessage.waitFor({ state: 'visible' });
  }

  async getBenchmarkByName(name: string): Promise<Locator> {
    return this.page.locator(`[data-testid="benchmark-item"]:has-text("${name}")`);
  }
}
```

## Usage in Tests

```typescript
import { test, expect } from '@playwright/test';
import { setupAuthenticatedTest } from '../helpers/auth';
import { BenchmarkPage } from '../pages/BenchmarkPage';

test('should create benchmark', async ({ page }) => {
  await setupAuthenticatedTest(page, {
    apiKey: process.env.TEST_API_KEY!,
    backendUrl: process.env.BACKEND_URL || 'http://localhost:3002',
    frontendUrl: process.env.FRONTEND_URL || 'http://localhost:3000',
  });

  const benchmarkPage = new BenchmarkPage(page);

  await benchmarkPage.createBenchmark('Test Benchmark');

  const benchmark = await benchmarkPage.getBenchmarkByName('Test Benchmark');
  await expect(benchmark).toBeVisible();
});
```

## Best Practices

### ✅ DO
- Use descriptive names for locators and methods
- Prefer `data-testid` attributes for selectors
- Use semantic selectors (role, label, text) when data-testid isn't available
- Keep methods focused on page interactions
- Document complex methods with JSDoc comments
- Return `Locator` objects from getter methods (don't await)

### ❌ DON'T
- Don't include assertions in Page Objects (keep them in tests)
- Don't hardcode test data in Page Objects
- Don't use CSS/XPath selectors unless absolutely necessary
- Don't duplicate locators across multiple Page Objects
- Don't include business logic

## Naming Convention

- File: `{PageName}Page.ts` (e.g., `BenchmarkPage.ts`)
- Class: `{PageName}Page` (e.g., `export class BenchmarkPage`)
- Use PascalCase for class names
- Use camelCase for methods and properties

## Generated Files

Page Objects in this directory may be automatically generated by the `/test-generator` skill based on:
- Feature documentation in `feature-docs/{feature}/playwright/`
- Page exploration data from `/playwright-explorer` skill
- Selectors documented in `*.selectors.md` files
